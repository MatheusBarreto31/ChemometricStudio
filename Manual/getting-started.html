Install and Run

Chemometric Studio is available as pre-built packages for Windows, macOS, and Linux, or you can run from source code for the latest development version.

Option 1: Pre-built Packages (Recommended for Production Use)

The releases contain stable, tested versions of the program. Visit our GitHub page at http://www.chemometricstudio.com to download the latest release for your platform.

Windows

  1. Download the ZIP archive labeled ChemometricStudio-*-Windows-*-x86_64.zip
  2. Extract the archive to your desired location.
  3. Open ChemometricStudio.exe for the full specialized version, or ChemometricStudioUser.exe for the user-friendly simplified version.

macOS

  1. Identify your processor:
    • Apple Silicon (M1, M2, M3, etc.): Download ChemometricStudio-*-macOS.AppleSilicon-*-arm64.tar.gz
    • Intel processor: Download ChemometricStudio-*-macOS.Intel-*-x86_64.tar.gz
  2. Extract the archive using Terminal or by double-clicking it.
  3. Navigate to the extracted ChemometricStudio folder and double-click either ChemometricStudio for the full version or ChemometricStudioUser for the user version.
  4. Note: On first run, you may need to grant permissions in System Preferences or use chmod +x to make the executable runnable.

Linux

Release packages are available for both x86_64 and ARM64 architectures. Download the package format that matches your Linux distribution and system architecture.

Debian/Ubuntu-based systems
  1. Download ChemometricStudio-*-Linux.Debian-based-*-<arch>.deb
  2. Install using your package manager: 
    sudo dpkg -i ChemometricStudio-*-Linux.Debian-based-*-<arch>.deb
  3. Launch from your application menu, or run ChemometricStudio for the full version and ChemometricStudioUser for the user version.
Red Hat/Fedora/CentOS-based systems
  1. Download ChemometricStudio-*-Linux.Fedora-*-<arch>.rpm
  2. Install using your package manager: 
    sudo dnf install ChemometricStudio-*-Linux.Fedora-*-<arch>.rpm
    or for older systems: 
    sudo yum install ChemometricStudio-*-Linux.Fedora-*-<arch>.rpm
  3. Launch from your application menu, or run ChemometricStudio for the full version and ChemometricStudioUser for the user version.
Arch Linux
  1. Download ChemometricStudio-*-Linux.Arch-based-*-<arch>.pkg.tar.zst
  2. Install using pacman: 
    sudo pacman -U ChemometricStudio-*-Linux.Arch-based-*-<arch>.pkg.tar.zst
  3. Launch from your application menu, or run ChemometricStudio for the full version and ChemometricStudioUser for the user version.

Option 2: Run from Source Code (Bleeding Edge Development)

For the latest development features (which may contain bugs), clone the source code and run directly using Python.

  1. Clone the repository: 
    git clone https://github.com/your-org/chemometric-studio.git
cd chemometric-studio
  2. Select your branch:
    • main: More stable, periodically tested
    • canary: Bleeding edge, latest features
    git checkout main  # or git checkout canary
  3. Create and activate a Python environment:
    • Windows: 
      python -m venv .venv
.\.venv\Scripts\Activate.ps1
    • macOS/Linux: 
      python -m venv .venv
source .venv/bin/activate
  4. Install dependencies: 
    pip install -r requirements.txt
  5. Start the application: 
    python launcher.py

First Pipeline in 5 Steps

  1. Open the Setup tab and add a data input function.
  2. Add preprocessing and model functions to Methodology in execution order.
  3. Select each function and fill required parameters (hover for help).
  4. Click Run Model to execute the pipeline.
  5. Open Analysis to inspect model outputs and create reports.

Files You Will See

  • model.json: generated pipeline definition.
  • model_log.txt: model run logs.
  • error_log.txt: error details from failed runs.

Step 1: Learn the Main Work Area (Setup)

This annotated screenshot highlights the controls you will use first.

  1. Main tabs: move between Setup, Analysis, Routing, and Report.
  2. Function Library: browse categories and add functions to your pipeline.
  3. List of Tasks: verify execution order before running.
  4. Task actions: reorder or remove functions from Methodology.
  5. Model execution and storage: run, load, and save your model.

Step 2: Configure Each Selected Function

After adding functions, click each item in the Methodology list and complete its required inputs.

  • The highlighted tab confirms you are in Setup.
  • The arrow points to the selected function instance in Methodology.
  • The right panel updates to show parameters for that selected function.
Tip: work from top to bottom in Methodology so upstream outputs are ready for downstream functions.

Step 3: Review Outputs in Analysis

Run your model, then switch to Analysis to inspect tables, plots, and navigation controls.

  • Analysis tab: opens calculated outputs for each function.
  • Page controls: move through generated analysis pages.
  • Navigation menus: browse multi-dimensional views and slices.
