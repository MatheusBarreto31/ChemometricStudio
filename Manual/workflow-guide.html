Setup Tab

The Setup tab is where you build the methodology pipeline by selecting and configuring functions.

Basic Workflow

  1. Add Functions: Click category names in the left panel to expand available functions.
  2. Add to Methodology: Click on a function to add an instance to your Methodology list (right side).
  3. Configure: Select each instance and fill in its required parameters in the lower panel.
  4. Review Execution Order: Functions execute top-to-bottom in the Methodology list—ensure data flows logically.
  5. Save: Use File → Save Model to persist your pipeline as model.json.

Typical Function Order

  1. Load Data (or Validation Data)
  2. Preprocessing (Baseline, Smoothing, Center & Scale)
  3. Modeling (PLS, PCA, PARAFAC)

Most routing is handled automatically. See Routing Tab for advanced scenarios.

Routing Tab (Advanced)

Most users can skip this section. The system automatically routes function outputs to inputs where parameter names match. Only use the Routing tab if you need to:

  • Override automatic routing behavior
  • Route outputs to non-obvious inputs
  • Debug complex workflows with many parallel branches

How Routing Works

When you add functions to your methodology, the system automatically connects outputs from one function to inputs of downstream functions based on parameter names. For example:

  • Load Data outputs X_cal → Baseline Correction receives X_cal
  • Baseline Correction outputs X_cal → Next function receives X_cal

When to Use Manual Routing

  1. Identify the function whose output you want to route (source).
  2. Identify the function that should receive that output (destination).
  3. Select the specific output parameter name.
  4. Select the destination function input parameter.
  5. Create the route and verify it displays in the routing map.
For typical workflows (sequential data input → preprocessing → modeling), manual routing is not required.

Execution Controls

  • Run Model: full methodology execution.
  • Run to here: run only up to selected function instance.
  • Timing Report in Tools menu: detailed runtime breakdown after execution.
