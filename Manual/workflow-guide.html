Setup Tab

The Setup tab is where you build and configure the methodology pipeline.

Checklist

  1. Browse function categories in the left panel.
  2. Add functions to Methodology in execution order.
  3. Select each instance and provide required parameters.
  4. Save the model periodically using File -> Save Model.

Best Practice

Configure inputs and preprocessing first. Then add model and validation steps. This makes routing and debugging easier.

Routing Tab

Routing defines exactly how data moves between functions.

  1. Pick a source function output.
  2. Pick the destination function input.
  3. Create the route and confirm it appears in the map.
  4. Repeat until all required inputs are connected.
If the model fails quickly, check for missing routes first.

Execution Controls

  • Run Model: full methodology execution.
  • Run to here: run only up to selected function instance.
  • Timing Report in Tools menu: detailed runtime breakdown after execution.
