User GUI

The User GUI is a simplified interface designed for non-specialized users who need guided model execution with minimal setup complexity.

It lets teams share validated models from expert users to operators, technicians, and collaborators who mainly need to load data, run the model, and inspect predefined outputs.

Why It Matters

  • Reduces misuse: hides advanced configuration that could break validated workflows.
  • Improves consistency: users run the same prepared model with the same expected logic.
  • Speeds onboarding: non-specialists can execute analyses without learning full model construction.
  • Supports operations: ideal for routine QC and production checks based on approved models.

When To Use It

  • You already have a model prepared by a specialist in the full interface.
  • Users need guided execution, not model architecture editing.
  • You want standardized outputs for recurrent decisions.
  • You need reliable day-to-day operation with lower training effort.

How To Use the User GUI

  1. Start the user application executable (ChemometricStudioUser).
  2. Load a pre-configured model supplied by the specialist team.
  3. Select the required input data files.
  4. Run the model and wait for execution to complete.
  5. Review guided outputs and follow your local decision protocol.
  6. Export or report results when required by your process.
If the model or inputs fail validation, send the model file and logs to the specialist team instead of editing internals in the user interface.

Good Practices

  • Use versioned model files approved by your organization.
  • Keep input templates and naming conventions consistent.
  • Document who ran each analysis and when.
  • Escalate unexpected results to the model owner before changing process settings.
