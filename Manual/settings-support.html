Tools Menu

The Tools menu provides diagnostic and reporting utilities to help you inspect execution behavior and troubleshoot workflow issues.

Available Tools

  • Timing Report — View a detailed breakdown of how long each step in your workflow took to execute. Useful for identifying performance bottlenecks and optimizing long-running analyses.
  • Execution Report — Review a comprehensive log of all messages, warning and errors raised by the executed functions during the run. Helps verify any mishaps during the model run.
  • Model Log — Access the application's internal log for the last model run. Contains detailed technical information about model operations, inputs, outputs, routing, etc, which can help diagnose unexpected behavior and debugging.

Using These Tools

  1. Run your workflow to generate execution data.
  2. Open the Tools menu from the menubar.
  3. Select the report or log you want to view — it will open in a popup window.
  4. Review the information; reports and logs can be used to verify expected behavior or troubleshoot issues.

When These Tools Are Available: Tools become available only after a workflow execution. If a tool is grayed out (disabled), run your workflow first to populate the necessary data. Each report captures information from the most recent run.

Troubleshooting Checklist

  1. Verify required function parameters in Setup.
  2. Verify routing completeness in Routing tab.
  3. Use Run to here for fault isolation.
  4. Inspect model_log.txt and error_log.txt.
  5. Recreate model to reproduce and validate fixes.
When reporting issues, include model file and relevant logs.
